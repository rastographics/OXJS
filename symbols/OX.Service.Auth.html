Overview

Namespace for auth related services.

In order to allow any JID to perform a command for a SIP address or subscribe to PubSub data regarding a SIP address or SIP domain the JID must authorize itself (or another JID) by providing a SIP user address and a password to the Auth component.

Extends:
OX.Base
See:
Auth Component
Requires:
A 'connectionAdapter' property inherited from an OX.Connection.

Method Detail

(void) OX.Service.Auth.authorizePlain(address, password, callbacks, options)

Authorize a JID for a SIP address, authorized via a password. This password is sent in clear text to the XMPP API, so your connection should be encrypted for your own safety.
Parameters:
(String) address
The SIP address for authentication.
(String) password
The web password for the SIP address.
(Object) callbacks Optional
An object supplying functions for 'onSuccess', and 'onError'.
(Object) options Optional
(String) options.jid
The JID to authorize for the SIP address. If unspecified, use the current JID from the underlying connection.
(String) options.authForAll
Authorize this JID for all SIP addresses associated with it.
Returns:
(void)

  ox.Auth.authorizePlain('sip@example.com', 'password', {
    onSuccess: function () {},
    onError:   function (error) {}
  }, { jid: 'jid@example.com', authForAll: true });
